Source: https://styles.refero.design/style/4363070d-02da-4954-88e4-d4a2101c5204  
Site: https://huggingface.co  
North star: High-contrast data console, a brightly lit control panel with precise readouts and subtle depth  
Theme: light

## Typography

- Primary family: Source Sans Pro.
- Primary fallbacks: Open Sans, Lato.
- Weights: 400, 600, 700.
- Sizes observed: 10px through 96px.
- Monospace family: IBM Plex Mono.
- Monospace fallbacks: Space Mono, Fira Code.
- Technical label size: 15px.

## Layout Intelligence

- High contrast is expressed as a dense but readable console, not a poster.
- Max width around 1280px.
- Default radius is 8px; large CTAs can use a pillier 25.6px radius.
- Component gaps are compact: 8px to 16px.
- Subtle shadows are allowed, but they remain shallow and utilitarian.
- Accent colors appear in badges, links, and small semantic marks rather than huge surfaces.

## Components

| Component | Role | Treatment |
| --- | --- | --- |
| Data Card | Repeated content | White or gray surface, 8px radius, subtle border/shadow |
| Link | Navigation/action | Azure blue, legible, sparingly used |
| Technical Label | Code/data cue | IBM Plex Mono, small and precise |
| Badge | State or category | Small saturated mark, contained color |
| Console Section | Dense UI area | Light background, sharp readouts, careful dividers |

## Rules Worth Carrying Forward

- High contrast can support dense information if tokens are disciplined.
- Use black for primary readability, gray for structure, blue for real interaction.
- Keep shadows extremely subtle.
- Use monospace only where it clarifies technical information.
- Avoid using every accent color as a theme; accents must remain semantic and contained.
