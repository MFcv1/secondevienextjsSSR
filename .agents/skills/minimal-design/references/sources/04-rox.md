# Rox - Minimal Design Source Notes

## Core read

Rox is the "analytical blueprint on pure white" archetype: a spacious,
near-achromatic financial automation interface where clarity comes from
typography, neutral hierarchy, and sparing blueprint-blue action. It has a
large, elegant display voice, but the actual UI stays geometric, compact, and
restrained.

This source teaches minimalism through typography and data clarity: dramatic
headline scale, almost no decoration, and subtle status colors only where data
needs them.

## Typography

- Hero display: FH Total Display Regular or Playfair Display fallback.
- Hero sizes: 106px and 183px with 0.8 line height.
- Body/UI: Geist or Inter fallback.
- Body sizes: 14-28px.
- Letter spacing around -0.02 on Geist.
- Reserve display face for hero-level moments only.

## Structure

- Compact density.
- 4px base unit.
- Card padding: 16px.
- Element gap: 4-16px.
- Default radius: 6px.
- Buttons: 8px.
- Large radius: 12px.
- Pill radius: 100px.
- Minimal shadows only.

## Minimal lessons

1. Huge type can be minimal
   - The headline is dramatic, but the rest is restrained.

2. Blue is the only primary chromatic action
   - CTA and active states only.

3. Hierarchy uses neutrals first
   - Do not use color to rank every text element.

4. Shadows are barely there
   - Small 0px/1px/2px shadows only.

5. Data colors stay semantic
   - Small status chips can use red/orange/yellow/violet.
   - These do not become brand accents.

## Anti-patterns

- Multiple saturated primary accents.
- Generic system font for hero.
- Heavy shadows.
- Color-coded hierarchy everywhere.
- Too many radii.
- Decorative imagery.
