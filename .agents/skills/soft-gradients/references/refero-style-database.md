<a id="philosophy"></a>
## 1. CORE PHILOSOPHY

Soft Gradients is not "add pastel blobs behind a hero." It is a lighting system
for interfaces. The gradient should feel like atmosphere, reflection, warmth,
or focus. It should make the product feel more alive while preserving the hard
work of usability: contrast, hierarchy, density, and affordance.

The most important rule:

**The gradient is light. The interface is still the object.**

A good soft-gradient page has three layers:

| Layer | Job | Typical treatment |
| --- | --- | --- |
| Ambient field | Emotional temperature and spatial depth | Low-saturation gradients, radial washes, warm/cool balance |
| Product surface | Usable cards, panels, inputs, command centers | White, pearl, graphite, navy, or charcoal surfaces with clear borders |
| Action signal | Conversion, focus, active state, intelligence | One controlled accent: lime, violet, blue, coral, amber, or orange |

If the gradient layer starts competing with the product surface, the style
breaks. If the action signal is spread everywhere, the page becomes noisy. If
the foreground text is pastel, the page becomes weak.

Soft Gradients should feel:

- luminous
- calm
- precise
- warm or ethereal
- modern without shouting
- optimistic without becoming childish
- premium without turning into empty luxury
- product-led instead of background-led

Soft Gradients should not feel:

- random
- sugary
- washed out
- generic purple SaaS
- decorative for its own sake
- low contrast
- all-glass everything
- "AI magic" with no workflow

The source set gives five usable poles:

| Source | Pole | What it teaches |
| --- | --- | --- |
| Base44 | Pearl AI canvas | Off-white warmth, lime action, soft gradient as product intelligence |
| Dia Browser | Ethereal glass search | Lavender/peach/rose ambient fields, frosted inputs, spacious calm |
| Sprig | Warm insight SaaS | Cream canvas, peach gradient, navy text, blue CTA discipline |
| Superhuman | Warm dusk productivity | Dark cinematic gradients, serif/sans polish, premium workflow focus |
| Retool | Obsidian ember command | Dark developer surfaces, amber glow, dense utility with warmth |

The range matters. Soft Gradients is not one palette. It can be light pearl,
frosted pastel, warm cream, dark dusk, or black ember. The common thread is
controlled luminosity.

### The Central Tension

Soft-gradient design is caught between two risks:

- Too soft: pretty background, weak hierarchy, illegible text, no product
  credibility.
- Too technical: sterile SaaS grid with a decorative blob pasted behind the
  hero.

The solution is to make every visual decision serve a role:

- gradients create atmosphere and focus
- surfaces create readability and hierarchy
- typography creates authority and calm
- accents create action and memory
- layout creates workflow
- motion creates gentle state change, not spectacle

### Practical Design Laws

1. Start with a neutral canvas.
2. Pick one emotional temperature: pearl, lavender, peach, sunset, or ember.
3. Pick one action accent.
4. Create one primary gradient language.
5. Keep product surfaces flatter than the background.
6. Put real UI in the foreground.
7. Use soft shadows and borders to separate, not to decorate.
8. Make body text dark, white, or high-contrast neutral.
9. Never let the gradient decide the content hierarchy.
10. Stop adding glow before the page starts looking magical everywhere.

<a id="when-to-use"></a>
## 2. WHEN TO USE THIS SKILL

Use Soft Gradients when a user asks for:

- soft gradients
- gentle gradient design
- atmospheric SaaS
- modern AI design
- pastel but premium
- luminous UI
- warm product design
- frosted glass interface
- soft-glow hero
- calm futuristic style
- browser/search assistant mood
- productivity with warmth
- AI dashboard with softness
- dark UI with warm gradients
- less sterile enterprise design

It works especially well for:

- AI builders
- browser tools
- search products
- productivity apps
- customer insight platforms
- feedback/research SaaS
- internal tool platforms
- workflow automation tools
- email/calendar products
- creative operating systems
- developer command centers
- premium B2B landing pages
- onboarding flows
- empty states
- product-led hero sections

Avoid this skill when:

- the product needs brutalist harshness
- the brand must feel fully monochrome
- the user specifically asks for no gradients
- the interface is mostly data tables with strict compliance requirements
- the style should be retro, grunge, handmade, or editorial print-first
- the visual identity depends on flat color blocks rather than light
- strong imagery or photography should dominate the page

This skill can combine with:

| Pairing | Result |
| --- | --- |
| Technical Sans | More precise AI/product workspace |
| Clean SaaS | Safer B2B adoption |
| Dark UI | Dusk or ember command-center style |
| Vibrant Accents | Stronger CTA and brand memory |
| Pastel | Softer consumer feel, if contrast is maintained |
| Editorial Minimal | More refined typography and narrative spacing |

Do not combine with too many styles at once. Soft Gradients already has a
strong atmosphere layer. Pair it with one structural style, not five.

<a id="archetypes"></a>
## 3. THE 5 SOURCE ARCHETYPES

### 3.1 Base44 - Pearl AI Canvas

**Mood:** warm off-white product intelligence.

Base44 uses a pearl canvas, clean sans typography, graphite text, and small
lime/orange accents. The gradients feel like ambient light around an AI tool,
not like a decorative rainbow. The page should read as a product first and a
soft visual identity second.

Key tokens:

| Role | Token |
| --- | --- |
| Page | Canvas Pearl `#faf9f7` |
| Surface | Snowdrift White `#ffffff` |
| Text | Ink Black `#000000`, Graphite Text `#232529` |
| Muted text | Slate Gray `#324158`, Stone Whisper `#696f7b` |
| Border | Ash Border `#cfcfcf` |
| Action | Lime Spritz `#ade900` |
| Soft action wash | Light Lime `#ebffb1` |
| Warm accent | Sunset Orange `#d8723c`, Blazing Orange `#ff631f` |

What to copy:

- pearl background instead of cold white
- one unusual AI action color
- compact product cards
- thin borders
- white surfaces on warm canvas
- gradients as limited light fields
- real UI modules in the hero

What to avoid:

- lime as broad background
- vague AI blob art
- childish green/yellow palette
- overly rounded toy cards
- low-contrast gray copy

Best for:

- AI app builders
- agent platforms
- product creation tools
- friendly SaaS landing pages
- "build something quickly" workflows

### 3.2 Dia Browser - Ethereal Gradient Glass

**Mood:** spacious browser/search magic.

Dia Browser uses frost white, lavender, peach, amber, and rose to create an
airy field around a central search/product interface. Its strength is the
relationship between a soft background and clear foreground controls.

Key tokens:

| Role | Token |
| --- | --- |
| Background | Frost White `#F9FAFB` |
| Ambient wash | Lavender Haze `#F3E8FF`, Peach Glow `#FDE68A`, Rose Quartz `#FECACA` |
| Text | Deep Ink `#1F2937` |
| Muted text | Slate Gray `#6B7280` |
| Border | Pearl Border `#E5E7EB` |
| Surface | Pure White `#FFFFFF` |
| Accent | Violet `#8B5CF6`, Amber `#F59E0B`, Rose `#F43F5E` |

What to copy:

- large soft-lit input/search field
- frosted white panels
- airy spacing
- low-saturation gradient fields
- pill controls with visible state
- gradient text used sparingly

What to avoid:

- white text on pastel
- excessive blur
- saturated purple-blue generic hero
- every card being glass
- gradient with no product object

Best for:

- browsers
- assistants
- search tools
- creative productivity
- onboarding
- calm futuristic pages

### 3.3 Sprig - Warm Insight SaaS

**Mood:** friendly intelligence for B2B.

Sprig mixes warm cream, peach/pink/yellow gradient washes, dark navy text, and
confident blue CTAs. It shows how soft gradients can support a serious
business product without becoming cold enterprise software.

Key tokens:

| Role | Token |
| --- | --- |
| Canvas | Warm Cream `#FFF7ED` |
| Surface | Pure White `#FFFFFF` |
| Text | Deep Navy `#0F172A` |
| Muted text | Medium Slate `#64748B`, Light Slate `#94A3B8` |
| Border | Border Gray `#E2E8F0` |
| Primary action | Accent Blue `#3B82F6` |
| Support accents | Purple `#8B5CF6`, Pink `#EC4899`, Orange `#F97316`, Green `#10B981` |
| Gradient | Peach Sunset `#FED7AA -> #FECACA -> #FDE68A` |

What to copy:

- navy text on warm canvas
- blue CTA discipline
- white insight cards
- gradient around intelligence moments
- mono labels for data precision
- friendly warmth without beige overload

What to avoid:

- all accent colors competing
- cream that becomes dull beige
- pastel body text
- enterprise dashboards with fluffy decoration only
- heavy black shadows on warm cards

Best for:

- research tools
- feedback platforms
- customer insight SaaS
- AI analysis products
- survey/analytics workflows

### 3.4 Superhuman - Warm Dusk Productivity

**Mood:** premium productivity at sunset.

Superhuman brings soft gradients into a darker, cinematic environment. Navy and
charcoal backgrounds support coral, amber, purple, and blue glows. It can use a
display serif for premium tone, but the product interface remains focused and
minimal.

Key tokens:

| Role | Token |
| --- | --- |
| Deep background | Midnight Navy `#0B1426` |
| Dark surfaces | Deep Charcoal `#111827`, Slate Blue `#1F2937` |
| Text | Pearl White `#F9FAFB`, Pure White `#FFFFFF` |
| Muted text | Warm Gray `#6B7280`, Soft Gray `#9CA3AF` |
| Warm glow | Sunset Coral `#FB7185`, Warm Amber `#F59E0B` |
| Support accents | Soft Purple `#A78BFA`, Electric Blue `#3B82F6`, Success Green `#10B981` |
| Gradients | Dusk and Sunset gradients |

What to copy:

- dark hero with one warm light source
- premium editorial headline treatment
- focused productivity cards
- mixed dark/light sections
- coral-to-amber CTA with restraint
- strong foreground contrast

What to avoid:

- red/orange alert feeling
- generic navy SaaS
- gray text too low on dark backgrounds
- too much serif inside UI
- cinematic empty hero with no workflow

Best for:

- email tools
- calendar/workflow products
- productivity platforms
- executive SaaS
- AI assistant pages
- premium conversion pages

### 3.5 Retool - Obsidian Ember Command Center

**Mood:** warm machinery for builders.

Retool uses black, charcoal, graphite, and amber/orange glow to make a dense
developer interface feel alive. The warm gradient is localized: hero focus,
CTA, hover, active panel, or focal aura. It never sacrifices readability for
atmosphere.

Key tokens:

| Role | Token |
| --- | --- |
| Background | Obsidian Black `#000000`, Charcoal `#1a1a1a` |
| Surface | Dark Charcoal `#171717`, Deep Graphite `#252525` |
| Text | Off White `#ededed`, Pure White `#ffffff` |
| Muted text | Muted Gray `#737373`, Light Gray `#a3a3a3` |
| Warm accent | Ember Orange `#ff6b35`, Amber Glow `#ff8c42` |
| Semantics | Error Red `#ef4444`, Success Green `#22c55e` |

What to copy:

- dense panels and real tool UI
- black-to-charcoal depth
- warm ember focal glow
- modest radii
- tight developer typography
- semantic status discipline

What to avoid:

- cyberpunk neon overload
- orange behind code/table text
- glass cards everywhere
- decorative dashboard panels with no function
- purple gradient drift

Best for:

- internal tools
- developer platforms
- AI ops dashboards
- observability tools
- command centers
- workflow builders

<a id="shared-dna"></a>
## 4. SHARED DESIGN DNA

Across all five references, the style has consistent principles even when the
palette changes.

### 4.1 Gradients behave like light

The gradient is not a sticker. It should imply:

- sunrise warmth
- ambient reflection
- screen glow
- atmospheric depth
- product intelligence
- dusk light
- ember heat
- soft optimism

Good gradients have a plausible source:

- behind the hero product object
- at the edge of the viewport
- behind a search box
- behind a key CTA zone
- radiating from a primary card
- emerging from a dark command center

Bad gradients float randomly with no relationship to content.

### 4.2 Foreground stays crisp

Soft does not mean blurry foreground. The foreground should be:

- readable
- cleanly aligned
- bordered where needed
- higher contrast than the background
- precise in spacing
- clear in interaction state

Use blur for depth. Do not use blur to avoid designing the card.

### 4.3 One accent leads

Each archetype has many possible colors, but one action signal wins:

| Archetype | Lead accent |
| --- | --- |
| Base44 | Lime |
| Dia Browser | Violet or amber |
| Sprig | Blue |
| Superhuman | Coral/amber |
| Retool | Ember orange |

The lead accent should handle:

- primary CTA
- active nav
- selected state
- focus ring
- important badge
- primary gradient stop

Other colors should support, not compete.

### 4.4 Typography is calm

Soft gradients need typography that can hold authority. Common choices:

- Inter-like sans for product clarity
- Space Grotesk-like display for future softness
- light serif display for premium warmth
- mono only for technical labels and data

Avoid:

- cartoon rounded fonts
- ultra-thin body text
- aggressive condensed type
- too many font families
- low-contrast pastel headings

### 4.5 Surfaces do the work

The best soft-gradient pages still have real interface surfaces:

- search bars
- cards
- sidebars
- dashboards
- command palettes
- email panels
- workflow builders
- insight modules
- feature matrices
- CTA strips

The gradient sets the atmosphere. The surfaces create trust.

### 4.6 Spacing is generous, but product areas can be dense

Landing sections can be spacious. Product modules can be compact. This contrast
is important:

- generous outside
- precise inside
- soft background
- crisp foreground
- airy narrative
- dense workflow

Do not make every element huge. Soft-gradient design fails when it becomes
only hero-scale mood.

## 5. COLOR SYSTEMS

Soft-gradient palettes need more discipline than flat palettes because the
background already contains multiple colors. Build the palette by role, not by
collecting colors you like.

### 5.1 Palette Architecture

Every palette should contain:

1. Canvas neutral
2. Surface neutral
3. Primary text
4. Secondary text
5. Border
6. Lead accent
7. Accent wash
8. Gradient stop A
9. Gradient stop B
10. Semantic success/warning/error

Example role map:

| Role | Light pearl | Ethereal | Warm SaaS | Dusk | Ember |
| --- | --- | --- | --- | --- | --- |
| Canvas | #faf9f7 | #F9FAFB | #FFF7ED | #0B1426 | #000000 |
| Surface | #ffffff | #ffffff | #ffffff | #111827 | #171717 |
| Text | #232529 | #1F2937 | #0F172A | #F9FAFB | #ededed |
| Muted | #696f7b | #6B7280 | #64748B | #9CA3AF | #a3a3a3 |
| Border | #cfcfcf | #E5E7EB | #E2E8F0 | rgba(255,255,255,.12) | #252525 |
| Lead | #ade900 | #8B5CF6 | #3B82F6 | #FB7185 | #ff6b35 |
| Warm stop | #ff631f | #F59E0B | #F97316 | #F59E0B | #ff8c42 |
| Soft stop | #ebffb1 | #F3E8FF | #FED7AA | #A78BFA | #fbbf24 |

### 5.2 Canvas Neutrals

Soft-gradient pages rarely work on default pure white alone. Use a canvas
neutral with temperature:

- pearl: `#faf9f7`
- frost: `#F9FAFB`
- cream: `#FFF7ED`
- dusk navy: `#0B1426`
- obsidian: `#000000`

The canvas should be quieter than the gradient. If it has too much color, the
whole page becomes tinted and text contrast becomes harder.

### 5.3 Surface Neutrals

Surfaces must be calmer than the gradient:

- white cards on warm/pearl backgrounds
- translucent white cards on ethereal backgrounds
- charcoal cards on obsidian backgrounds
- navy cards on dusk backgrounds
- graphite panels inside dark command centers

Surface fills should have controlled opacity:

```css
.surface-pearl {
  background: rgba(255, 255, 255, 0.82);
  border: 1px solid rgba(31, 41, 55, 0.08);
  box-shadow: 0 24px 80px rgba(31, 41, 55, 0.08);
}

.surface-frosted {
  background: rgba(255, 255, 255, 0.68);
  border: 1px solid rgba(255, 255, 255, 0.72);
  backdrop-filter: blur(22px);
  box-shadow: 0 30px 90px rgba(139, 92, 246, 0.12);
}

.surface-dusk {
  background: rgba(17, 24, 39, 0.82);
  border: 1px solid rgba(249, 250, 251, 0.12);
  box-shadow: 0 40px 120px rgba(0, 0, 0, 0.42);
}
```

### 5.4 Accent Rules

Pick one primary accent:

- lime for AI creation energy
- violet for ethereal future/search
- blue for trustworthy B2B intelligence
- coral/amber for premium warmth
- orange for developer command energy

Then assign support accents:

| Accent | Good role | Bad role |
| --- | --- | --- |
| Lime | primary AI action, generated state | broad background, body text |
| Violet | selected state, future glow | every icon, every CTA, all gradients |
| Blue | primary B2B CTA, link | random decorative blob |
| Pink | warm highlight, insight category | main body text |
| Orange | ember action, warm highlight | alert substitute everywhere |
| Amber | glow edge, secondary warmth | low-contrast text |
| Green | success | brand accent unless intentional |
| Red | error/destructive | generic excitement |

### 5.5 Gradient Stop Discipline

Use 2-4 stops. More stops usually look amateur unless the stops are very close
in saturation and luminance.

Good stop structures:

```css
/* Pearl AI */
linear-gradient(135deg, #ebffb1 0%, #c7f9ff 52%, #ffffff 100%);

/* Ethereal glass */
linear-gradient(135deg, #F3E8FF 0%, #FDE68A 48%, #FECACA 100%);

/* Warm insight */
linear-gradient(135deg, #FED7AA 0%, #FECACA 50%, #FDE68A 100%);

/* Dusk productivity */
linear-gradient(135deg, #0B1426 0%, #1F2937 52%, #FB7185 100%);

/* Ember command */
radial-gradient(circle at 75% 15%, rgba(255, 107, 53, .34), transparent 34%),
linear-gradient(180deg, #000000 0%, #171717 100%);
```

Bad stop structures:

```css
/* Too much unrelated hue */
linear-gradient(90deg, red, orange, yellow, green, blue, purple);

/* Too saturated for soft-gradient UI */
linear-gradient(135deg, #ff00ff, #00ffff);

/* Low contrast if used behind text */
linear-gradient(135deg, #f9d5ff, #e7fff4);
```

### 5.6 Contrast Rules

Minimum practical rules:

- Body on light: use `#0F172A`, `#1F2937`, `#232529`, or equivalent.
- Muted body on light: stay darker than `#6B7280` when copy is important.
- Body on dark: use `#F9FAFB`, `#ededed`, or equivalent.
- Muted body on dark: avoid going below `#9CA3AF` for paragraph text.
- Pastel colors are backgrounds, not body text.
- Gradients behind text need an overlay or a separate solid surface.

<a id="typography"></a>
## 6. TYPOGRAPHY SYSTEMS

Soft gradients need typography with enough clarity to keep the page from
feeling vague. Treat type as the stabilizer.

### 6.1 Recommended Pairings

| Mood | Primary | Secondary | Notes |
| --- | --- | --- | --- |
| Pearl AI | Inter, Satoshi, system-ui | JetBrains Mono | Modern and pragmatic |
| Ethereal search | Inter | Space Grotesk | Gentle future, not sci-fi |
| Warm insight | Inter | JetBrains Mono | B2B clarity plus data precision |
| Dusk productivity | Inter | Crimson Text or editorial serif | Premium, human, refined |
| Ember command | Inter/system | SF Mono, JetBrains Mono | Developer precision |

### 6.2 Type Scale

Use a flexible but controlled scale:

| Token | Size | Use |
| --- | --- | --- |
| xs | 12px | labels, metadata |
| sm | 14px | compact UI text |
| base | 16px | body |
| lg | 18px | lead body, card intro |
| xl | 20px | compact heading |
| 2xl | 24px | card section heading |
| 3xl | 30px | page subheading |
| 4xl | 36px | hero support heading |
| 5xl | 48px | hero heading |
| 6xl | 64px | major hero heading |
| 7xl | 72px | rare premium display |

### 6.3 Weight Rules

- Light display can work on pearl or dusk pages, but body must stay readable.
- Use 400 for most copy.
- Use 500 or 600 for UI labels and card headings.
- Use 700 only when the visual identity needs firmness.
- Avoid ultra-black 900 weight unless paired with very minimal gradients.

### 6.4 Tracking Rules

- Display: `-0.02em` to `-0.03em` only when the font supports it.
- Body: `0`.
- Buttons: `0` or very slight positive only for uppercase labels.
- Do not use negative tracking on small UI text.

### 6.5 Line-Height Rules

| Text | Line height |
| --- | --- |
| Hero display | 0.98-1.12 |
| Large heading | 1.08-1.18 |
| Card heading | 1.15-1.25 |
| Lead paragraph | 1.45-1.6 |
| Body | 1.55-1.7 |
| Compact UI | 1.25-1.4 |

### 6.6 Typographic Voice

Soft-gradient copy should be confident and precise. It should not over-explain
the visual effect.

Good:

- "Build internal tools from a prompt."
- "Search, summarize, and act across every tab."
- "Turn customer signals into product decisions."
- "The fastest inbox for focused teams."
- "Ship workflows with production-grade controls."

Bad:

- "A magical luminous journey into the future of your productivity."
- "Experience vibrant vibes and soft pastel dreams."
- "Beautiful gradients for everything you do."

### 6.7 Serif Usage

Use serif only when the page needs premium human warmth. It is strongest for:

- a hero headline
- a testimonial pull quote
- a section title
- a founder/editorial note

Do not use serif for:

- buttons
- tables
- navigation labels
- dense product UI
- form labels
- code/log interfaces

### 6.8 Mono Usage

Mono adds precision when used sparingly:

- metric labels
- event timestamps
- keyboard hints
- model names
- code snippets
- status IDs
- command palette rows

Do not set the whole page in mono unless the user explicitly wants a technical
monospace style.

## 7. SPACING, SHAPE, AND DENSITY

Soft-gradient pages need enough whitespace for ambient light to breathe, but
too much whitespace can make the interface feel empty. Balance spacious
sections with dense product modules.

### 7.1 Base Unit

Use a 4px base unit.

```css
:root {
  --space-1: 4px;
  --space-2: 8px;
  --space-3: 12px;
  --space-4: 16px;
  --space-5: 20px;
  --space-6: 24px;
  --space-8: 32px;
  --space-10: 40px;
  --space-12: 48px;
  --space-16: 64px;
  --space-20: 80px;
  --space-24: 96px;
}
```

### 7.2 Density Modes

#### Comfortable Product

Use for Base44 and Sprig-like pages.

- section gap: 48-72px
- card padding: 24-40px
- element gap: 12-20px
- nav height: 56-72px
- button height: 40-48px

#### Ethereal Spacious

Use for Dia-like pages.

- section gap: 80-112px
- hero vertical padding: 96-140px
- card padding: 24-32px
- element gap: 16-24px
- input height: 56-72px

#### Premium Dusk

Use for Superhuman-like pages.

- section gap: 88-120px
- hero vertical padding: 110-160px
- card padding: 28-40px
- element gap: 20-28px
- max width: 1120-1240px

#### Dense Command

Use for Retool-like pages.

- section gap: 56-88px
- card padding: 16-28px
- panel padding: 12-20px
- row height: 36-48px
- toolbar height: 40-52px

### 7.3 Radius Scale

Soft Gradients often uses rounded shapes, but radius must match the product
personality.

| Token | Radius | Use |
| --- | --- | --- |
| sharp | 3-4px | precise buttons, developer UI |
| sm | 8px | inputs, compact cards |
| md | 12px | buttons, pills, general UI |
| lg | 16px | premium cards, modal panels |
| xl | 24px | frosted cards, ethereal panels |
| pill | 999px | tags, segmented controls |

Rules:

- Pearl AI: 4-14px.
- Ethereal glass: 12-24px.
- Warm insight: 8-16px.
- Dusk productivity: 8-16px.
- Ember command: 4-12px.

Do not use 32px radius on every card unless the brand is intentionally soft and
consumer-oriented.

### 7.4 Layout Width

Recommended max widths:

| Layout | Max width |
| --- | --- |
| Hero content | 720-980px |
| Full product showcase | 1040-1280px |
| Editorial section | 760-900px |
| Feature grid | 1120-1280px |
| Command center mockup | 1180-1440px |
| Dense dashboard | 1280-1520px |

### 7.5 Alignment

Soft gradients can become visually loose. Use strict alignment:

- align card edges
- align buttons to text baselines
- align icons inside square containers
- use grid columns consistently
- center only when the content is truly symmetrical
- left-align dense product and B2B sections

### 7.6 Foreground Ratio

The page should not be all atmosphere. A useful rule:

- 55-70% neutral surface/text
- 20-35% ambient background
- 5-10% accent and glow

For dark ember command centers:

- 75-85% dark neutral
- 10-18% warm glow
- 3-7% semantic colors

<a id="gradient-grammar"></a>
## 8. GRADIENT GRAMMAR

The gradient system is the heart of this skill. Build it with grammar: source,
shape, strength, color, and relationship to content.

### 8.1 Gradient Sources

Every gradient should have an implied source:

| Source | Use |
| --- | --- |
| Horizon | warm hero sections, sunset productivity |
| Orb field | soft AI aura, but keep it subtle |
| Edge light | ambient color entering from viewport edge |
| Product glow | glow behind a card or device mockup |
| Input halo | search/assistant focus |
| Ember core | dark command center focus |
| Section wash | background warmth for a feature band |

### 8.2 Gradient Shapes

Use these shapes:

#### Linear ambient wash

Best for warm SaaS and pearl AI.

```css
.ambient-linear {
  background:
    linear-gradient(135deg, rgba(254, 215, 170, .55), rgba(254, 202, 202, .42) 45%, rgba(253, 230, 138, .35)),
    #FFF7ED;
}
```

#### Radial focal glow

Best behind a product card.

```css
.ambient-radial {
  background:
    radial-gradient(circle at 50% 22%, rgba(139, 92, 246, .22), transparent 34%),
    radial-gradient(circle at 72% 18%, rgba(245, 158, 11, .18), transparent 30%),
    #F9FAFB;
}
```

#### Edge light

Best for a page that should feel immersive without blobs in the center.

```css
.ambient-edge {
  background:
    radial-gradient(circle at 100% 0%, rgba(251, 113, 133, .28), transparent 32%),
    radial-gradient(circle at 0% 20%, rgba(167, 139, 250, .20), transparent 30%),
    #0B1426;
}
```

#### Ember field

Best for dark developer pages.

```css
.ambient-ember {
  background:
    radial-gradient(circle at 72% 12%, rgba(255, 107, 53, .28), transparent 30%),
    radial-gradient(circle at 88% 40%, rgba(251, 191, 36, .12), transparent 24%),
    linear-gradient(180deg, #000 0%, #171717 100%);
}
```

### 8.3 Gradient Strength

Use opacity and stop position deliberately:

| Intensity | Opacity | Use |
| --- | --- | --- |
| Barely there | 0.05-0.10 | section warmth |
| Ambient | 0.12-0.22 | background light |
| Focal | 0.24-0.38 | behind hero object |
| CTA | 0.85-1.00 | button or badge fill |
| Warning | 1.00 | semantic state only |

If the gradient is behind text, reduce opacity or insert a surface.

### 8.4 Gradient Placement

Good placements:

- top-right hero edge
- behind the product preview
- inside a large empty hero zone
- behind a search/input module
- as a subtle section background
- inside CTA strip
- on active button fill
- as a 1px border highlight

Bad placements:

- directly behind paragraphs
- under tables
- behind code
- behind small UI labels
- on every card background
- as page-wide rainbow bands

### 8.5 Gradient Borders

Soft-gradient borders can create premium depth:

```css
.gradient-border {
  position: relative;
  border-radius: 24px;
  background: linear-gradient(#fff, #fff) padding-box,
              linear-gradient(135deg, rgba(139,92,246,.45), rgba(245,158,11,.32), rgba(244,63,94,.26)) border-box;
  border: 1px solid transparent;
}
```

Use for:

- hero product card
- special insight module
- primary dashboard frame
- premium testimonial

Do not use for:

- every feature card
- navigation
- dense table rows
- secondary buttons

### 8.6 Mesh-Like Backgrounds Without Blob Slop

If you need mesh feeling, use large low-opacity radial gradients with clear
positions:

```css
.mesh-soft {
  background:
    radial-gradient(900px circle at 12% 8%, rgba(243, 232, 255, .85), transparent 42%),
    radial-gradient(760px circle at 86% 18%, rgba(253, 230, 138, .62), transparent 38%),
    radial-gradient(680px circle at 55% 88%, rgba(254, 202, 202, .52), transparent 40%),
    #F9FAFB;
}
```

Rules:

- use large radii
- keep opacity low
- avoid hard stop bands
- do not animate too fast
- keep text on surfaces
- avoid saturated cyan/magenta unless cyber neon is intended

### 8.7 Gradient Text

Gradient text is risky. Use only for:

- one word in hero
- one number in a metric
- one badge label
- a short brand phrase

```css
.gradient-word {
  color: transparent;
  background: linear-gradient(135deg, #8B5CF6, #F59E0B, #F43F5E);
  -webkit-background-clip: text;
  background-clip: text;
}
```

Never set long paragraphs in gradient text.

### 9.2 Blur Rules

Blur works when it supports depth. It fails when it becomes a default style.

Use blur for:

- one hero search panel
- modal overlay
- translucent nav on a gradient field
- background light diffusion
- decorative ambient layers

Avoid blur for:

- every feature card
- body text containers
- dense dashboard panels
- code/log surfaces
- table rows

Practical blur scale:

| Token | Blur | Use |
| --- | --- | --- |
| sm | 8px | subtle nav translucency |
| md | 16px | frosted small panels |
| lg | 22px | hero glass card |
| xl | 32px | decorative background only |

### 9.3 Shadow Systems

Soft-gradient shadows should be diffused. Avoid harsh black drop shadows on
light pages.

Light shadows:

```css
--shadow-soft: 0 18px 60px rgba(31, 41, 55, .08);
--shadow-frost: 0 30px 90px rgba(139, 92, 246, .14);
--shadow-warm: 0 24px 80px rgba(249, 115, 22, .12);
```

Dark shadows:

```css
--shadow-dusk: 0 40px 120px rgba(0, 0, 0, .46);
--shadow-ember: 0 28px 90px rgba(0,0,0,.52), 0 0 48px rgba(255,107,53,.12);
```

Rules:

- On light pages, shadow should often be colored by the gradient family.
- On dark pages, shadow can be black, but glow must be limited.
- A border plus subtle shadow is usually better than shadow alone.
- Do not use large hard shadows under every card.

### 9.4 Surface Layering

Use 3-5 elevation levels:

| Level | Light | Dark |
| --- | --- | --- |
| 0 | canvas | obsidian/navy |
| 1 | soft wash section | charcoal section |
| 2 | white card | graphite card |
| 3 | frosted/raised card | elevated dark panel |
| 4 | modal/focus object | glowing command object |

Never nest card inside card inside card without clear reason. If a tool needs
dense modules, use panel groups and dividers rather than decorative card stacks.

## 13. MOTION AND INTERACTION

Soft-gradient motion should feel like light shifting or focus changing. It
should not feel like a carnival.

### 13.1 Motion Principles

- slow ambient movement
- quick UI response
- small hover elevation
- subtle glow changes
- focused input halo
- gentle card reveal
- no distracting infinite shimmer on content

### 13.2 Timing

| Interaction | Duration | Easing |
| --- | --- | --- |
| Button hover | 120-180ms | ease-out |
| Card hover | 160-220ms | ease-out |
| Modal/card reveal | 220-320ms | cubic-bezier(.2,.8,.2,1) |
| Ambient gradient drift | 12-28s | linear or ease-in-out |
| Focus halo | 120-180ms | ease-out |

### 13.3 Hover Patterns

Light card hover:

```css
.soft-card {
  transition: transform 180ms ease, box-shadow 180ms ease, border-color 180ms ease;
}

.soft-card:hover {
  transform: translateY(-2px);
  border-color: rgba(139,92,246,.18);
  box-shadow: 0 24px 70px rgba(31,41,55,.10);
}
```

Dark panel hover:

```css
.dark-panel {
  transition: transform 160ms ease, border-color 160ms ease, box-shadow 160ms ease;
```

### 13.4 Ambient Gradient Drift

Use sparingly. Respect reduced motion.

```css
.ambient-drift {
  background-size: 140% 140%;
  animation: drift 22s ease-in-out infinite alternate;
}

@keyframes drift {
  from { background-position: 0% 40%; }
  to { background-position: 100% 60%; }
}

@media (prefers-reduced-motion: reduce) {
  .ambient-drift {
    animation: none;
  }
}
```

Rules:

- Do not animate behind long reading sections.
- Do not animate gradients on every card.
- Keep motion slow enough that users do not consciously track it.
- Avoid animating `filter: blur()` heavily.

### 13.5 Focus States

Soft-gradient focus states should be visible and elegant:

```css
.input:focus-visible {
  outline: 0;
  border-color: rgba(139,92,246,.44);
  box-shadow:
    0 0 0 4px rgba(139,92,246,.12),
    0 18px 60px rgba(139,92,246,.14);
}
```

For ember:

```css
.control:focus-visible {
  outline: 0;
  border-color: rgba(255,107,53,.55);
  box-shadow: 0 0 0 4px rgba(255,107,53,.14);
}
```

### 13.6 Loading States

Prefer:

- soft pulse on skeletons
- subtle shimmer only inside product preview
- progress ring with accent color
- generation state with lime/violet/blue glow

Avoid:

- full-page rainbow loaders
- aggressive shimmer across paragraphs
- glowing skeletons that hurt readability

<a id="anti-slop"></a>
## 14. ANTI-SLOP RULES

This section is strict. Use it to prevent generic AI design slop.

### 14.1 Gradient Slop

Reject:

- random blobs behind every section
- saturated purple-blue gradient as default
- gradients directly behind body text
- rainbow stop lists
- hard color bands
- tiny radial gradients that look like stickers
- mesh background with no relationship to content
- gradient on every button, card, icon, and headline

Replace with:

- one controlled ambient field
- one focal glow
- one accent gradient
- neutral surfaces
- clear content hierarchy

### 14.2 Glassmorphism Slop

Reject:

- every card is translucent
- blur on dense UI
- glass panels with unreadable text
- nested frosted cards
- no border distinction
- high blur over busy background

Replace with:

- one hero glass object
- solid content cards
- borders for separation
- blur only where depth matters

### 14.3 Pastel Slop

Reject:

- pastel body text
- candy-store palette
- low-contrast buttons
- childish pink/yellow/blue everywhere
- cream backgrounds with beige body copy

Replace with:

- dark navy or graphite text
- pastel as background/wash only
- one strong action accent
- accessible contrast

### 14.4 AI Slop

Reject:

- generic sparkles
- "magic" copy with no product detail
- glowing orbs as the main visual
- fake chat bubbles with no workflow
- impossible dashboards
- too many floating cards

Replace with:

- real prompt/input/product module
- assistant output with structure
- builder canvas
- workflow steps
- concrete benefit copy

### 14.5 Dark Gradient Slop

Reject:

- black background plus neon purple glow
- red/orange that feels like danger
- low-contrast gray text
- cyan-magenta cyberpunk when the ask is soft
- glowing code behind every section

Replace with:

- navy/charcoal depth
- coral/amber/ember warmth
- controlled glow around focal objects
- bright readable text
- semantic status colors

### 14.6 SaaS Slop

Reject:

- generic bento grid with pastel blobs
- "AI-powered platform" as only headline substance
- feature cards that are all identical
- blue CTA plus random gradient background
- oversized hero with no product screenshot

Replace with:

- specific product promise
- one real workflow preview
- cards with distinct jobs
- accent roles
- strong layout rhythm

### 14.7 Quick Audit Questions

Ask these before finishing:

1. Can I name the lead accent in one word?
2. Does every gradient have an implied light source?
3. Is body text high contrast?
4. Is the product object clearer than the background?
5. Are there more than two gradient treatments competing?
6. Is blur used in more than three component types?
7. Does the page still work if the gradient is removed?
8. Are semantic colors separate from brand colors?
9. Is the hero visual a real product/workflow object?
10. Does the style feel specific to the product category?

<a id="decision-tree"></a>
## 15. DECISION TREE

Use this tree when choosing a soft-gradient direction.

### 15.1 Product Category

If the product is an AI builder:

- choose Pearl AI Canvas
- canvas: pearl/off-white
- accent: lime or unusual AI action color
- visual: builder, generator, or workflow card
- avoid: cyber neon and vague magic

If the product is a browser/search assistant:

- choose Ethereal Gradient Glass
- canvas: frost white
- accent: violet/amber
- visual: search command input
- avoid: dense enterprise dashboard hero

If the product is customer insights or B2B intelligence:

- choose Warm Insight SaaS
- canvas: cream
- accent: blue
- visual: insight dashboard and white cards
- avoid: childish pastels

If the product is premium productivity:

- choose Warm Dusk Productivity
- canvas: navy/dark
- accent: coral/amber
- visual: inbox/calendar/workflow object
- avoid: alert-like red or empty luxury

If the product is developer/internal tools:

- choose Obsidian Ember Command
- canvas: black/charcoal
- accent: ember orange
- visual: dense builder or dashboard
- avoid: cyberpunk glow and low contrast code

### 15.2 Brand Personality

| Personality | Direction |
| --- | --- |
| Friendly AI | Pearl AI or Warm Insight |
| Calm futuristic | Ethereal Glass |
| Serious B2B | Warm Insight with restrained gradients |
| Premium and fast | Dusk Productivity |
| Technical and powerful | Ember Command |
| Creative and magical | Ethereal Glass with more lavender/rose |
| Enterprise but warm | Warm Insight or Ember Command |

### 15.3 Gradient Intensity

Choose intensity based on content:

- Marketing hero: ambient to focal.
- Product dashboard: low ambient.
- CTA section: focal or button gradient.
- Documentation: barely there.
- Command center: localized glow.
- Forms: focus halo only.
- Pricing: subtle section wash, no heavy gradient behind prices.

### 15.4 Light vs Dark

Pick light when:

- trust and accessibility matter most
- the product is B2B or productivity
- screenshots need clarity
- the target user is working all day
- the brand should feel optimistic

Pick dark when:

- the product is developer-focused
- the brand needs cinematic/premium impact
- the hero needs high drama
- the UI itself is command-center-like
- warm glow is part of the identity

Pick mixed when:

- a dark hero sells emotion
- light sections provide proof
- the product has both marketing and workflow density

### 18.6 Pricing Section

Soft gradients can make pricing feel warm, but prices need clarity.

Rules:

- Use solid cards.
- Use one highlighted plan with accent border or glow.
- Keep price text high contrast.
- Do not put price on a busy gradient.
- Use gradient only in header strip or selected-plan halo.


### 18.7 Testimonial Section

Soft-gradient testimonials work best with one large quote and a quiet surface.

Rules:

- Use warm wash behind quote.
- Put quote on solid or frosted surface.
- Use serif only for pull quote if desired.
- Keep attribution compact and sans.
- Do not scatter many colorful quote cards.

### 18.8 Feature Grid

Feature cards should not all have unique gradients.

Better:

- one shared neutral surface
- small accent icon per category
- one larger featured card with gradient
- consistent border and radius

Worse:

- six cards each with a different gradient
- pastel icon backgrounds plus gradient text plus glow
- no hierarchy among cards

```css
.cta-band {
```

- Use one CTA.
- Keep headline short.
- Do not hide the button in the gradient.
- Make the band feel like a culmination, not another hero.

- small ambient icon field
- clear explanation
- one action
- one secondary import/template action
- subtle accent

Avoid:

- huge illustration with no action
- low-contrast copy
- rainbow empty-state icon

<a id="checklist"></a>
## 20. QUALITY CHECKLIST

Use this checklist before delivering a soft-gradient design.

### 20.1 Identity

- [ ] The chosen archetype is clear.
- [ ] The lead accent has one primary role.
- [ ] The gradient language is specific, not generic.
- [ ] The palette has canvas, surface, text, border, accent, and semantic roles.
- [ ] The style matches the product category.

### 20.2 Contrast

- [ ] Body text passes practical contrast.
- [ ] Muted text is still readable.
- [ ] Buttons have clear text contrast.
- [ ] Text is not placed directly on busy gradient.
- [ ] Dark sections use bright enough text.

### 20.3 Gradients

- [ ] Every gradient has an implied light source.
- [ ] Gradients are not used on every component.
- [ ] Stop count is limited.
- [ ] Saturation is controlled.
- [ ] Ambient gradients do not overpower foreground UI.

### 20.4 Surfaces

- [ ] Foreground cards/panels are crisp.
- [ ] Blur is limited and purposeful.
- [ ] Borders are visible enough.
- [ ] Shadows are soft and diffused.
- [ ] Dense UI uses solid surfaces.

### 20.5 Typography

- [ ] Display type is authoritative.
- [ ] Body type is readable.
- [ ] Tracking is not negative on small text.
- [ ] Serif/mono usage is role-based.
- [ ] Copy is concrete, not vague magic language.

### 20.6 Layout

- [ ] Hero has a real product/workflow object.
- [ ] Sections have enough breathing room.
- [ ] Product modules have useful density.
- [ ] Alignment is strict.
- [ ] Responsive behavior preserves text and controls.

### 20.7 Interaction

- [ ] Hover states are subtle.
- [ ] Focus rings are visible.
- [ ] Motion respects reduced motion.
- [ ] Loading states are calm.
- [ ] Active states are unmistakable.

### 20.8 Anti-Slop

- [ ] No random pastel blobs.
- [ ] No low-contrast pastel text.
- [ ] No rainbow gradients.
- [ ] No glassmorphism everywhere.
- [ ] No generic AI sparkles as the main identity.
- [ ] No cyberpunk drift unless intended.

<a id="prompting"></a>
## 21. PROMPTING GUIDE

Use these prompts when instructing an agent or design system.

### 21.1 General Soft Gradients Prompt

Create a modern soft-gradient product interface where gradients behave like
controlled ambient light, not decoration. Use a high-contrast neutral text
system, crisp foreground surfaces, one primary accent color, and real product
UI in the hero. Avoid random blobs, pastel text, rainbow gradients,
glassmorphism on every element, and generic AI magic.

### 21.2 Pearl AI Prompt

Design a pearl/off-white AI product workspace inspired by a warm builder
canvas. Use white cards, graphite text, thin borders, a rare lime primary
action, subtle lime/cool gradient light behind the product preview, and compact
builder/workflow UI. The page should feel friendly, capable, and product-led.

### 21.3 Ethereal Search Prompt

Design an ethereal browser/search assistant interface with frost white canvas,
lavender/peach/rose ambient gradients, a large frosted command input, violet
focus states, amber warmth, and spacious typography. Keep all text on stable
surfaces and make the search workflow the central object.

### 21.4 Warm Insight Prompt

Design a warm B2B insight platform with cream background, navy text, blue
primary CTA, white dashboard cards, peach/pink/yellow gradient wash, and mono
data labels. The style should feel approachable but credible, with readable
metrics and clear product hierarchy.

### 21.5 Dusk Productivity Prompt

Design a premium productivity landing page with dark navy dusk gradients,
coral/amber warm light, crisp pearl text, a focused inbox/workflow product
panel, and a concise premium headline. Use optional serif display type only for
editorial warmth, not dense UI.

### 21.6 Ember Command Prompt

Design a dark developer command-center interface with obsidian background,
charcoal panels, off-white text, localized ember orange glow, dense builder UI,
tables/logs/toolbars, and clear semantic status colors. Keep gradients away
from dense text and avoid cyber neon.

### 21.7 Negative Prompt

Do not use generic purple-blue gradient blobs, rainbow mesh backgrounds,
pastel body text, low contrast labels, glass cards everywhere, stock vector
people, random sparkles, oversized empty hero sections, or gradients that hide
the actual product.

### Best Text Colors

| Background | Text |
| --- | --- |
| Pearl/cream/frost | #0F172A, #1F2937, #232529 |
| Pastel gradient | Put text on white/frosted surface |
| Navy/dusk | #F9FAFB, #ededed |
| Obsidian | #ededed, #ffffff |

### Best Gradient Placements

- behind hero product card
- around search input
- section wash
- CTA band
- active button fill
- edge light
- focal halo behind dashboard

### Worst Gradient Placements

- behind paragraphs
- under tables
- under code
- every card
- every icon
- page-wide rainbow
- inside long body text

### Build Order

1. Pick archetype.
2. Pick canvas and surface.
3. Pick lead accent.
4. Define one ambient gradient.
5. Define one focal glow.
6. Define typography.
7. Build real product object.
8. Add cards and sections.
9. Add hover/focus states.
10. Audit contrast and anti-slop.

### Final Taste Test

The page should still look credible with gradients turned off. With gradients
on, it should feel warmer, deeper, more specific, and more memorable. If the
gradient is doing all the work, the design is not finished.
