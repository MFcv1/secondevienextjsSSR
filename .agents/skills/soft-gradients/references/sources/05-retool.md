# Retool - Soft Gradients Source Notes

## Core read

Retool is the "obsidian ember command center" archetype: deep black and
charcoal product surfaces, warm amber/orange gradients, and a developer-tool
layout that remains dense, useful, and serious. It proves that a soft-gradient
system can live inside a utilitarian interface when the glow is controlled and
role-based.

The emotional texture is "warm machinery": command center, internal tools,
developer productivity, and enterprise craft. The orange is not playful; it is
the ember that keeps the dark UI alive.

## Typography

- Primary should be a precise sans, often system or Inter-like.
- Technical labels may use mono, but the main marketing copy can stay sans.
- The product UI should feel compact.
- Headings need enough weight to survive dark backgrounds.
- Labels, status chips, and code-like details can sit at 11-13px.

## Structure

- Dense product layout is allowed.
- Use panel groups, table/list rows, sidebars, toolbars, tabs, and console
  surfaces.
- Background should move from black to charcoal, not from black to purple.
- Amber gradients should be localized around hero focus, CTA, or active state.
- Radius can be modest; over-rounding weakens the developer-tool feel.

## Component lessons

1. Ember glow on command surface
   - Add warm light around the focal panel.
   - Avoid glowing every card.
   - Let the rest of the page stay utilitarian.

2. Dense dark panels
   - Multiple panels can coexist when borders and text hierarchy are clean.
   - Use graphite rather than pure black for cards.

3. Warm CTA in dark UI
   - Orange CTA can feel premium if it is not neon.
   - Hover can brighten toward amber.

4. Status palette discipline
   - Keep red and green purely semantic.
   - Amber is brand/action, not every warning.

5. Developer readability
   - Tables, code, charts, and logs need contrast first.
   - Gradients should never sit directly behind dense text.

## Reusable pattern

Use this source when building:

- internal tool platforms
- developer dashboards
- AI ops command centers
- observability/productivity tools
- dark enterprise landing pages
- command palettes and builders

## Anti-patterns

- Turning dark UI into neon cyberpunk.
- Orange glow behind long paragraphs.
- Soft gradients that reduce code/table readability.
- Too many rounded glass cards.
- Marketing-card layout instead of workflow layout.
