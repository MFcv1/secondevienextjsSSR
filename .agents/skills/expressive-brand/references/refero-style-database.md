<a id="philosophy"></a>
## 1. CORE PHILOSOPHY

Expressive Brand is not "make it colorful." It is the discipline of making a
site feel like it could only belong to this product.

The strongest expressive brands are not random. They are strict. They choose a
few brand levers and repeat them with confidence:

- a character vocabulary
- a proprietary type gesture
- a single vivid action color
- a dramatic dark-to-light transition
- a warm canvas nobody else uses
- a motion curve with personality
- a shape rule that never changes
- a specific illustration system
- a product frame treatment
- a recurring micro-brand detail

The key rule:

**Expression becomes brand only when it is repeatable.**

A mascot on the hero is not an identity. A rainbow palette is not an identity.
A weird font once is not an identity. A blob gradient behind a headline is not
an identity. The identity emerges when the same choices become a system:

- characters appear with the same anatomy
- labels use the same tracking
- buttons obey the same radius law
- one accent owns one job
- product shots use the same frame
- sections repeat a consistent rhythm
- motion has the same emotional timing

Expressive Brand should feel:

- memorable
- specific
- opinionated
- cohesive
- useful
- emotionally legible
- category-aware
- product-backed

It should not feel:

- random
- childish by accident
- trend-chasing
- decorative only
- overdesigned
- inaccessible
- disconnected from product reality
- like a moodboard pasted onto SaaS

The source set gives five poles:

| Source | Archetype | What it teaches |
| --- | --- | --- |
| Family | Illustrated fintech storybook | Character system as brand identity on disciplined cream UI |
| mymind | Private memory editorial | Single warm accent, serif voice, uppercase labels, pastel surface logic |
| Antimetal | Electric infrastructure signal | Technical drama, one chartreuse CTA, dark hero to light product proof |
| Empower | Bold finance utility | High-contrast yellow action, huge condensed type, dark/light trust rhythm |
| Retool | Warm obsidian workshop | Proprietary type, warm dark material, slab surfaces, shape grammar |

This skill is about building a distinct taste system, not a template. The
question is always:

**What is the minimum set of expressive moves that makes this brand impossible
to confuse with a generic competitor?**

### The Expressive Brand Equation

Use this equation:

```txt
Brand Voice = Product Category + Audience Emotion + Visual Lever + Repetition Rule
```

Examples:

```txt
Family = finance + approachability + characters + cream storybook system
mymind = memory + privacy + serif/orange editorial + label/dot repetition
Antimetal = infrastructure + urgency + electric signal + dark-to-light product transition
Empower = finance + confidence + yellow/condensed type + dark utility contrast
Retool = developer tools + craft + warm obsidian type + slab/pill shape grammar
```

If a design lacks one part of the equation, it weakens:

- Product category missing: pretty but irrelevant.
- Audience emotion missing: visually interesting but cold.
- Visual lever missing: generic.
- Repetition rule missing: random.

### The Three Levels of Expressiveness

#### Level 1: Accent Expression

One color, shape, or type detail becomes memorable.

Use when:

- product must stay conservative
- content is dense
- audience expects clarity
- brand just needs differentiation

Example moves:

- yellow CTA on black
- orange nav dots
- chartreuse pill action
- warm off-white canvas

#### Level 2: System Expression

Multiple coordinated levers form a recognizable identity.

Use when:

- brand wants to be remembered
- category is crowded
- product has enough proof UI
- marketing page can carry personality

Example moves:

- serif headlines + uppercase orange labels + pastel cards
- dark hero + technical globe + chartreuse CTA + light dashboard
- warm obsidian + compressed type + slab surfaces + binary radius

#### Level 3: World Expression

The entire site feels like a visual world.

Use when:

- brand is consumer-facing or creative
- illustration is part of identity
- product needs emotional reframing
- user asks for personal/creative/unique design

Example moves:

- illustrated characters surrounding headline
- full brand object vocabulary
- motion personality
- custom hero composition
- recurring character/prop system

<a id="when-to-use"></a>
## 2. WHEN TO USE THIS SKILL

Use Expressive Brand when the user asks for:

- creative identity
- strong brand
- memorable site
- unique visual direction
- personal design
- product with character
- modern but specific
- playful professional
- bold fintech
- high-taste brand system
- not generic SaaS
- custom visual identity
- distinctive category design
- brandable UI
- strong art direction
- expressive typography
- custom illustrations or mascot-like elements

It works especially well for:

- fintech products that need warmth
- crypto or wallet products that need approachability
- personal knowledge tools
- AI products that need emotional differentiation
- developer platforms that need craft
- internal tool platforms that need a recognizable voice
- B2B SaaS in crowded markets
- creative software
- brand launches
- studio portfolios
- campaign landing pages
- consumer productivity
- product-led marketing pages

Use with caution for:

- medical systems
- government services
- legal compliance flows
- very dense dashboards
- crisis/incident workflows
- enterprise procurement pages
- products where brand playfulness could reduce trust

Avoid this skill when:

- user wants plain utility
- brand must be intentionally invisible
- product needs pure documentation
- visual identity is already fixed and strict
- there is no time to build a coherent system
- accessibility requirements forbid the chosen expressive lever

Pair with:

| Pairing | Result |
| --- | --- |
| Technical Sans | Expressive but credible SaaS |
| Editorial Minimal | More refined typographic personality |
| Vibrant Accents | More color and conversion energy |
| Soft Gradients | More ambient emotional atmosphere |
| Dark UI | More dramatic and premium |
| Playful Design | More character-driven consumer voice |
| High-End Design | More luxury and restraint |

Do not combine too many expressive systems. Choose one main lever and one
supporting lever.

## 3. THE 5 SOURCE ARCHETYPES

### 3.1 Family - Illustrated Fintech Storybook

**Mood:** warm, playful, trustworthy, adventurous.

Family proves that illustration can be the identity system. The page is built
on a warm cream canvas, tactile flat cards, dark pill CTA, tight friendly
display type, and vivid characters that surround the headline. It makes finance
and crypto feel approachable without losing structure.

Key tokens:

| Role | Token |
| --- | --- |
| Canvas | `#fbfaf9` |
| Card/ring | `#f2f0ed`, `#f8f7f4` |
| Text | `#474645`, `#343433`, `#121212` |
| Primary CTA | `#121212` with white text |
| Primary brand accent | `#ff3e00` |
| Illustration colors | `#00ca48`, `#0090ff`, `#ffbb26`, `#ff58ae`, `#9f4fff` |

What to copy:

- illustration as system, not decoration
- warm cream canvas
- dark pill CTA as anchor
- flat cards with inset borders
- expressive character vocabulary
- headline surrounded by characters
- tight custom display type used sparingly

What to avoid:

- random mascot stickers
- pure white canvas
- orange filled CTA
- too many character colors in one scene
- drop shadows on all cards
- playful font in dense UI

Best for:

- fintech
- consumer crypto
- wallets
- kids/family finance
- community products
- approachable technical products

### 3.2 mymind - Private Memory Editorial

**Mood:** private, reflective, warm, personal.

mymind uses one warm orange accent across nav dots, eyebrow labels, outline
buttons, and gradient atmosphere. It pairs editorial serif headlines with
clean sans UI and pastel feature cards. It feels personal and calm rather than
loud.

Key tokens:

| Role | Token |
| --- | --- |
| Warm accent | `#ff5924` |
| Accent gradient | `linear-gradient(#ff5924, #ffa86a)` |
| Link | `#1573dd` |
| Text | `#000000`, `#24272d`, `#3a475a`, `#4a5465` |
| Muted | `#717286`, `#748297` |
| Canvas | `#f9fafc`, `#ffffff` |
| Pastel cards | `#e5eaf2`, `#f3f0e7`, `#dde9d3` |

What to copy:

- single accent used in many micro roles
- serif display voice
- uppercase tracked orange labels
- product frames with thin borders
- pastel cards without shadows
- dark manifesto section for emotional contrast

What to avoid:

- filled orange buttons
- multiple saturated accents
- generic sans-only page
- heavy shadows on tinted cards
- fast mechanical motion

Best for:

- personal productivity
- notes/memory tools
- bookmarking/collection apps
- private AI spaces
- writing/knowledge products

### 3.3 Antimetal - Electric Infrastructure Signal

**Mood:** technical, dramatic, precise, high-signal.

Antimetal uses a dark electric hero, then snaps into a light product UI. A
single chartreuse CTA bridges both contexts. The expression is infrastructural:
dot-matrix globe, blue glow, technical dashboard cards, tight tracking, and
blue-tinted shadows.

Key tokens:

| Role | Token |
| --- | --- |
| Dark structural blue | `#001033`, `#1b2540` |
| Hero atmosphere | navy/electric-blue/cyan gradient |
| Primary CTA | `#d0f100` |
| Hero ghost detail | `#e0f6ff` |
| Light canvas | `#f8f9fc` |
| Surface | `#ffffff` |
| Muted technical text | `#596075`, `#6b7184`, `#7c8293` |
| Border | `#b1b5c0` |

What to copy:

- single dramatic dark hero
- immediate light product proof
- chartreuse only as CTA
- technical illustration as atmosphere
- serif display for high-craft infrastructure
- blue-tinted shadow/ring system
- universal pill buttons

What to avoid:

- chartreuse decoration
- multiple dark bands
- black drop shadows
- non-pill buttons
- cyber neon overload

Best for:

- infrastructure SaaS
- devops
- cloud cost tools
- observability
- security
- technical AI systems

### 3.4 Empower - Bold Finance Utility

**Mood:** direct, confident, useful, high-contrast.

Empower uses deep dark sections, a single vivid yellow action color, strong
condensed/expanded typography, rounded cards, and sharp transitions to white
content sections. It shows how expressive brand can be built from type and a
single action color instead of illustration.

Key tokens:

| Role | Token |
| --- | --- |
| Dark canvas | `#100f0f`, `#171616` |
| Dark card | `#262525` |
| Muted dark text/border | `#64635c` |
| Warm light | `#fffdf6`, `#ffffff` |
| Primary accent | `#e4e24e` |
| Soft accent | `#faf9b6` |

What to copy:

- yellow as only saturated accent
- enormous assertive type
- condensed/expanded type voice
- dark/light section contrast
- large friendly card radii
- no-nonsense utility copy

What to avoid:

- extra accent colors
- generic system font headlines
- heavy shadows
- decorative graphics that weaken directness
- low-contrast yellow text

Best for:

- finance
- insurance
- banking
- payroll
- benefits
- high-trust utilities

### 3.5 Retool - Warm Obsidian Workshop

**Mood:** crafted, technical, warm, proprietary.

Retool builds expression through warm black, parchment text, large compressed
light-weight headings, slab surfaces, binary radius grammar, muted teal washes,
and a hero gradient that feels like heat in a workshop. It is expressive but
not colorful.

Key tokens:

| Role | Token |
| --- | --- |
| Canvas | `#151515` |
| Receding surface | `#0e0e0e` |
| Raised slab | `#242424` |
| Border | `#3f403d` |
| Muted | `#8b867f`, `#94958e`, `#b6b8af`, `#cbccc4` |
| Primary text/action | `#e9ebdf` |
| Teal wash | `#185849`, `#0e352c` |
| Hero atmosphere | amber/rust radial gradient |
| Shimmer | parchment/coral/steel-blue phrase highlight |

What to copy:

- warm dark canvas
- huge light compressed type
- no-shadow slab surfaces
- parchment as bright color
- pill or square, no middle radius
- chromatic colors only in special effects

What to avoid:

- pure black
- neon accents
- generic bold Inter
- heavy drop shadows
- mixed random radii
- saturated UI chrome

Best for:

- developer platforms
- internal tools
- workflow builders
- technical brands needing craft
- dark B2B landing pages

<a id="shared-dna"></a>
## 4. SHARED DESIGN DNA

### 4.1 Expression is role-based

Every expressive element has a job:

- illustration reframes the category
- typography creates voice
- accent color guides action
- warm canvas sets emotional temperature
- dark hero creates drama
- product UI proves credibility
- motion gives personality
- shape grammar makes components recognizable

If an element has no job, it is decoration. Decoration is allowed only when it
belongs to a repeatable visual vocabulary.

### 4.2 Product proof balances brand personality

The more expressive the hero, the more concrete the product proof must be.

| Expressive move | Product proof needed |
| --- | --- |
| Mascots and characters | Real UI mockup, card system, clear CTA |
| Editorial serif and orange atmosphere | Product screenshot frames and feature cards |
| Dark electric hero | Light dashboard immediately after |
| Huge condensed finance type | Clear benefits, trust content, action flow |
| Warm obsidian workshop | Real builder panels, integrations, tool surfaces |

Do not build an expressive hero and then continue with vague generic cards.

### 4.3 One lever should dominate

Pick one main expressive lever:

- illustration
- typography
- color
- contrast mode
- material
- motion
- composition

Then pick one support lever.

Good combinations:

- illustration + warm canvas
- serif type + orange accent
- dark hero + chartreuse CTA
- condensed type + yellow action
- warm dark material + compressed type

Bad combinations:

- illustration + serif + neon + gradient + huge type + mascot + 3D objects
- multiple unrelated accent colors
- every section with a new style
- every card with a unique shape

### 4.4 Repetition creates ownership

Repeat the brand move across scales:

| Scale | Example |
| --- | --- |
| Hero | characters, huge type, dark signal, warm gradient |
| Section | labels, cards, surface colors, repeated shapes |
| Component | buttons, badges, rings, product frames |
| Micro | nav dots, icon fills, label tracking, hover states |

The strongest identities appear in tiny UI details, not only the hero.

### 4.5 Category tension makes the brand interesting

Expressive brands often work by bending category expectations:

- crypto becomes storybook
- personal memory becomes editorial
- infrastructure becomes electric and high-craft
- finance becomes bold and optimistic
- developer tools become warm workshop craft

Ask:

```txt
What emotion is missing from this category?
```

Then build a system around that missing emotion.

<a id="brand-levers"></a>
## 5. BRAND LEVERS

### 5.1 Illustration Vocabulary

Use when the brand needs warmth, approachability, emotional reframing, or
consumer memory.

Define:

- character anatomy
- object vocabulary
- stroke rules
- fill palette
- placement rules
- motion rules
- max character density
- relationship to text

Example system:

```txt
Characters:
- organic blob bodies
- stick limbs
- dot eyes
- simple smile/mouth curves
- one dominant color per character
- darker stroke/shadow variant per color

Objects:
- coins
- hearts
- stars
- locks
- arrows
- magnifying glass

Placement:
- flank headline
- overlap headline bounding box
- never cover functional UI
- max 4-5 colors in a scene
```

### 5.2 Signature Typography

Use when the brand needs proprietary voice without relying on illustration.

Typography can express:

- friendliness
- authority
- privacy
- confidence
- technical craft
- editorial warmth
- urgency
- directness

Rules:

- choose one display type gesture
- define exact size/weight/tracking
- keep display type out of dense UI
- pair with a calmer functional font
- repeat labels/captions consistently

Common gestures:

- friendly serif hybrid at hero scale
- editorial serif with italic variants
- extended/condensed finance display
- light compressed sans on dark
- technical UI sans with tight tracking

### 5.3 Single-Accent Discipline

Use when the brand needs memorability but cannot afford visual chaos.

Examples:

- Family: orange for links/illustration, not button fill
- mymind: orange for dots/eyebrows/outlines/atmosphere
- Antimetal: chartreuse only for CTA
- Empower: yellow for primary action
- Retool: parchment as bright UI color, chroma only in shimmer

Rules:

- define primary accent role
- define forbidden roles
- define support tints
- define semantic colors separately
- avoid using accent as decoration and CTA simultaneously unless intentional

### 5.4 Mode Transition

Use when the brand needs drama plus usability.

Pattern:

```txt
dramatic hero -> precise product daylight
dark identity -> light proof
emotional section -> functional section
cinematic claim -> dashboard evidence
```

Rules:

- make the transition intentional
- do it once or twice, not every section
- carry one token across modes
- keep typography consistent
- use product UI as the bridge

### 5.5 Shape Grammar

Use when components should feel ownable.

Examples:

- Family: 32px pill buttons, 10px cards, 72px illustration containers
- Antimetal: every button is pill, no exceptions
- Empower: big friendly 24px+ cards
- Retool: button is either pill or square, no medium radius

Rules:

- define radius by component type
- avoid arbitrary radii
- use shape to express personality
- preserve tap targets

### 5.6 Surface Grammar

Surfaces can express brand:

- warm cream flat cards
- pastel card tints
- blue-tinted technical shadows
- black/yellow finance blocks
- warm obsidian slabs

Rules:

- choose surface system before layout
- define elevation method
- avoid mixing shadow and border systems randomly
- use the same surface treatment across sections

### 5.7 Motion Personality

Motion is part of brand when it has a repeatable feeling:

- playful spring
- slow reflective deceleration
- technical overshoot snap
- direct utility ease
- mechanical deliberate motion

Rules:

- define base micro timing
- define reveal timing
- define hero/illustration timing
- avoid random easings
- respect reduced motion

## 6. COLOR SYSTEMS

Expressive Brand color needs a strategy. The mistake is adding personality by
adding many colors. Strong brands often use fewer colors more clearly.

### 6.1 Palette Architecture

Every expressive brand palette should contain:

1. canvas
2. primary text
3. muted text
4. surface
5. border/ring
6. primary accent
7. support tint
8. semantic colors
9. optional illustration colors
10. optional dark-mode surface

### 6.2 Warm Playful Palette

Use for Family-like systems.

```css
:root {
  --canvas: #fbfaf9;
  --surface: #f8f7f4;
  --ring: #f2f0ed;
  --text: #343433;
  --body: #474645;
  --muted: #848281;
  --cta: #121212;
  --cta-text: #ffffff;
  --orange: #ff3e00;
  --green: #00ca48;
  --blue: #0090ff;
  --yellow: #ffbb26;
  --pink: #ff58ae;
  --purple: #9f4fff;
}
```

Rules:

- warm canvas, not pure white
- dark CTA anchors conversion
- orange is link/illustration, not filled CTA
- illustration palette is separate from UI palette
- cards stay flat

### 6.3 Editorial Warm Accent Palette

Use for mymind-like systems.

```css
:root {
  --canvas: #f9fafc;
  --surface: #ffffff;
  --text: #000000;
  --nav: #24272d;
  --heading: #3a475a;
  --body: #4a5465;
  --muted: #717286;
  --border: #e2e2e2;
  --orange: #ff5924;
  --orange-soft: #ffa86a;
  --cobalt: #1573dd;
  --card-blue: #e5eaf2;
  --card-parchment: #f3f0e7;
  --card-sage: #dde9d3;
}
```

Rules:

- orange is outline/label/atmosphere
- cobalt is rare link color
- pastel cards differentiate content
- do not fill orange buttons

### 6.4 Electric Technical Palette

Use for Antimetal-like systems.

```css
:root {
  --navy: #1b2540;
  --cosmos: #001033;
  --chartreuse: #d0f100;
  --ice: #e0f6ff;
  --canvas: #f8f9fc;
  --surface: #ffffff;
  --storm: #596075;
  --slate: #6b7184;
  --ash: #7c8293;
  --border: #b1b5c0;
}
```

Rules:

- chartreuse is CTA only
- dark hero is singular
- light product canvas handles proof
- shadows/rings are blue-tinted

### 6.5 Bold Finance Palette

Use for Empower-like systems.

```css
:root {
  --night: #100f0f;
  --deep: #171616;
  --card-dark: #262525;
  --metal: #64635c;
  --cloud: #fffdf6;
  --white: #ffffff;
  --yellow: #e4e24e;
  --yellow-soft: #faf9b6;
}
```

Rules:

- yellow owns primary action
- no extra saturated colors
- dark and white sections alternate for clarity
- do not use yellow for low-contrast text

### 6.6 Warm Obsidian Palette

Use for Retool-like systems.

```css
:root {
  --canvas: #151515;
  --void: #0e0e0e;
  --slab: #242424;
  --rim: #3f403d;
  --copper: #8b867f;
  --ash: #94958e;
  --limestone: #b6b8af;
  --fog: #cbccc4;
  --parchment: #e9ebdf;
  --forest: #185849;
  --moss: #0e352c;
}
```

Rules:

- #151515 is the canvas, not pure black
- parchment is primary text/action
- teal is atmospheric wash, not generic accent
- saturated shimmer colors are special-effect only

### 6.7 Semantic Color Separation

Do not confuse expressive colors with semantic colors.

| Brand color | Semantic risk | Fix |
| --- | --- | --- |
| Family green | could look like success | define separate success if needed |
| mymind orange | could look like warning | keep warning more amber/red |
| Antimetal chartreuse | could look like success | reserve as CTA only |
| Empower yellow | could look like warning | warning needs icon/label |
| Retool teal | could look like success | use semantic green separately |

### 6.8 Accent Role Matrix

| Role | Family | mymind | Antimetal | Empower | Retool |
| --- | --- | --- | --- | --- | --- |
| Primary CTA | black pill | restrained outline/neutral | chartreuse pill | yellow pill | parchment pill |
| Links | orange/blue | cobalt/orange dots | navy/chartreuse only where action | yellow/dark | parchment/teal rare |
| Decoration | characters | orange atmosphere | blue glow/globe | minimal | amber-rust hero |
| Cards | cream/white | pastel tints | white technical cards | dark/white | dark slabs |
| Motion highlight | characters | slow pop | overshoot signal | direct ease | shimmer phrase |

## 7. TYPOGRAPHY SYSTEMS

Typography is often the most powerful expressive lever. Use it intentionally.

### 7.1 Type Pairing Models

#### Friendly Display + Functional Sans

Use for Family-like brands.

- Display: warm custom serif/hybrid, 44-68px.
- UI: Inter/system, 12-17px.
- Display appears rarely.
- UI remains practical.

#### Editorial Serif + Spaced Labels

Use for mymind-like brands.

- Display: serif, regular/italic.
- Labels: uppercase, positive tracking.
- Body: clean sans.
- Contrast creates emotional rhythm.

#### Craft Serif + Technical Sans

Use for Antimetal-like brands.

- Display: serif for high-craft technical headlines.
- UI: tight custom sans for dashboards.
- Product UI remains dense and technical.

#### Condensed/Expanded Utility Type

Use for Empower-like brands.

- Display: heavy, tight, wide/condensed custom face.
- Body/nav: extended sans.
- Labels: expanded with tracking.
- The type itself becomes the brand.

#### Light Compressed Sans

Use for Retool-like brands.

- Display: huge, light, negative tracking.
- Body: modest weights.
- Labels: positive tracking in micro font.
- Authority through scale, not bold weight.

### 7.2 Type Scale

Use a broad but controlled scale:

| Token | Size | Use |
| --- | --- | --- |
| micro | 11-12px | captions, tags, tiny labels |
| label | 13-14px | nav, badges, product labels |
| body-sm | 14-15px | compact card copy |
| body | 16-17px | paragraphs |
| body-lg | 18-21px | lead copy |
| heading-sm | 22-26px | card heads |
| heading | 32-40px | section heads |
| heading-lg | 44-48px | major section heads |
| display | 60-96px | hero |
| mega | 120-205px | rare brand moment |

### 7.3 Tracking Rules

Expressive brands often use tracking as signature:

- friendly display: negative tracking at hero scale
- editorial serif: negative display, positive labels
- technical sans: tight tracking everywhere
- finance display: tight and bold
- workshop labels: positive micro tracking

Do not use viewport-scaled font size. Use responsive clamps with fixed
min/max, not `vw` alone.

### 7.4 Font Role Rules

Every font needs a job:

| Font role | Use | Avoid |
| --- | --- | --- |
| Display font | hero/major sections | form labels, nav, tables |
| Body sans | readable copy | giant brand statement unless intentional |
| Mono/technical | code, IDs, technical labels | general paragraphs |
| Expanded/condensed | headlines, nav, labels | long body copy |
| Serif italic | emotional emphasis | all UI |

### 7.5 Fallback Guidance

When exact fonts are unavailable:

| Source | Fallback |
| --- | --- |
| Family display | Fraunces, Playfair Display |
| Family UI | Inter, IBM Plex Sans |
| mymind serif | Fraunces, Libre Baskerville, Cormorant Garamond |
| mymind labels | Nunito Sans, Inter |
| Antimetal display | Fraunces, Freight-like serif |
| Antimetal UI | Inter Variable, DM Sans |
| Empower display | Archivo Black, League Spartan, Poppins ExtraBold |
| Empower extended | Archivo Expanded if available, Inter Tight fallback |
| Retool display | Satoshi, Inter Tight, DM Sans |
| Retool labels | Space Grotesk, Geist |

### 7.6 Typographic Anti-Slop

Reject:

- generic Inter for everything when brand asks expressive
- decorative type used in all UI
- too many font families
- display type with no tracking control
- huge type that wraps badly on mobile
- uppercase labels with no letter spacing
- low contrast thin type on dark backgrounds

Replace with:

- one display voice
- one functional UI face
- one optional label/mono face
- clear tracking rules
- mobile line-break control

## 8. ILLUSTRATION AND GRAPHIC SYSTEMS

Illustration is powerful only when it is systematic.

### 8.1 Character System

Define:

- body shape
- limbs
- face
- stroke
- fill
- pose
- scale
- relationship to text
- allowed props
- animation

Family-like vocabulary:

```txt
Body:
- organic blob
- no hard geometry
- one dominant fill color
- darker shade for depth stroke

Face:
- dot eyes
- simple mouth curve
- optional eyebrow/cheek detail

Limbs:
- stick arms/legs
- simple rounded ends
- expressive pose

Props:
- coins
- hearts
- stars
- locks
- arrows
- magnifying glass
```

### 8.2 Illustration Placement

Good:

- surrounding hero headline
- overlapping headline bounding box lightly
- punctuating section transitions
- embedded inside product mockup contexts
- grouped around a product concept

Bad:

- random corner stickers
- repeating identical character
- covering buttons
- sitting behind long text
- replacing product proof

### 8.3 Illustration Color Rules

- 4-5 brand colors max per scene.
- Each character claims one color family.
- Do not place same-color characters adjacent.
- Use darker stroke/shadow variant per color.
- Keep UI colors separate from illustration colors.
- Avoid gradients unless the system explicitly uses flat gradients.

### 8.4 Graphic Systems Without Characters

Expressive brand can use non-character graphics:

- dot-matrix globe
- technical topology
- nav bullets
- orange section dots
- chartreuse signal
- warm radial atmosphere
- product screenshots in unique frames
- shimmer phrase
- slab surfaces

Define these as vocabulary too:

```txt
Technical signal vocabulary:
- dot matrix
- radial glow
- network arcs
- pill CTA
- blue-tinted shadow
- light product dashboard
```

### 8.5 Product Imagery Rules

Expressive brand still needs product reality:

- screenshots should be contained
- frames should repeat
- device mockups need consistent radius
- dashboard cards must be readable
- illustrations should not hide product proof
- product UI can carry brand tokens

#### Technical Compact

Use for Antimetal-like systems.

- base: 4px
- section gap: 80px
- card padding: 20px
- element gap: 8px
- dense dashboard modules
- product UI after hero

#### Bold Utility

Use for Empower-like systems.

- base: 8px

- element gap: 8px
- large heading blocks
- dark/light bands
- rounded modules

#### Workshop Spacious

Use for Retool-like systems.

- base: 4px

- large headline scale

| Archetype | Radius logic |
| --- | --- |
| Family | 10px cards, 24px mockups, 32px pills, 72px illustration containers |

Expressive pages need rhythm:

- hero identity
- product proof
- feature system
- emotional section

- conversion section

- hero, then six identical feature cards

Motion makes expression feel alive. Define it as a system.

### 12.1 Motion Archetypes

| Archetype | Motion personality |
| --- | --- |
| Family | playful spring and character bob |

| Empower | direct utility ease |
| Retool | deliberate mechanical precision |

### 12.2 Timing Scale

| Motion | Duration |
| --- | --- |
| Micro hover | 120-200ms |
| Button color/shape | 160-220ms |
| Card hover | 180-260ms |
| Section reveal | 400-700ms |
| Character entrance | 700-1000ms |
| Atmosphere animation | 1000ms+ or slow loop |

### 12.3 Easing Recipes

Playful:


Reflective:

```css
--ease-reflective: cubic-bezier(.19, 1, .22, 1);
```

Technical overshoot:

```css
--ease-signal: linear(
  0 0%,
  .026 1.8%,
  .108 3.9%,
  .59 12.2%,
  .792 16.5%,
  .931 21%,
  .978 23.4%,
  1.01 25.9%,
  1.033 29.3%,
  1.04 33.3%,
  1.001 56.9%,
  1 100%

```

Mechanical:

```css

```

Use for illustration systems:

- never distract from CTA

<a id="anti-slop"></a>
## 13. ANTI-SLOP RULES

### 13.1 Brand Slop

Reject:

- random colors with no role
- hero-only personality
- mascot stickers with no vocabulary
- one weird font used everywhere
- brand elements missing from components
- decorative graphics unrelated to product

Replace with:

- one main expressive lever
- repetition rules
- micro-brand details
- product proof
- reusable component grammar

### 13.2 Illustration Slop

Reject:

- inconsistent character anatomy
- too many colors per scene
- illustrations covering UI
- stock vector people
- random props
- gradients when the system is flat

Replace with:

- defined character vocabulary
- max color count
- prop library
- placement rules
- repeated motion behavior

### 13.3 Typography Slop

Reject:

- generic fonts for an expressive brief
- decorative font in paragraphs
- too many typefaces
- display type at small sizes
- poor mobile wrapping
- missing tracking rules

Replace with:

- display font for hero/sections
- functional font for UI
- optional label/mono font
- exact tracking and weights

### 13.4 Color Slop

Reject:

- multiple saturated CTAs
- accent used for everything
- semantic colors confused with brand colors
- random pastel surfaces
- bright colors on dark with no contrast check

Replace with:

- one primary accent
- support tints
- semantic separation
- strict forbidden roles

### 13.5 Product-Proof Slop

Reject:

- expressive hero with no product
- fake dashboard with meaningless UI
- illustration replacing actual workflow
- screenshots with no brand treatment
- product proof that looks from another website

Replace with:

- real product frame
- workflow states
- branded UI tokens inside product mockup
- consistent screenshot treatment

### 13.6 Motion Slop

Reject:

- random easing per component
- everything bouncing
- slow decorative animations on important UI
- shimmer everywhere
- no reduced-motion support

Replace with:

- defined personality curve
- limited motion vocabulary
- micro/reveal timing scale
- reduced-motion fallback

<a id="decision-tree"></a>
## 14. DECISION TREE

### 14.1 Choose By Category

If product is consumer finance/crypto:

- choose Illustrated Fintech Storybook
- use warm canvas, characters, dark CTA
- keep product UI serious

If product is personal memory/knowledge:

- choose Private Memory Editorial
- use serif, orange labels/dots, pastel cards
- keep tone reflective

If product is infrastructure/devops:

- choose Electric Infrastructure Signal
- use dark technical hero, chartreuse CTA, light dashboard
- keep drama concentrated

If product is financial utility:

- choose Bold Finance Utility
- use yellow CTA, huge type, dark/light contrast
- avoid playful decoration

If product is developer platform/workflow builder:

- choose Warm Obsidian Workshop
- use warm black, parchment text, slab surfaces, compressed type
- avoid cyber neon

### 14.2 Choose By Desired Emotion

| Emotion | Direction |
| --- | --- |
| approachable | characters + warm canvas |
| private | serif + warm accent |
| urgent/technical | electric dark hero + signal CTA |
| confident | bold type + yellow action |
| crafted | warm dark material + proprietary type |
| premium | restrained type + material surfaces |
| playful | illustration vocabulary |
| serious but distinct | single accent + shape grammar |

### 14.3 Choose Main Lever

Ask:

1. Is the audience consumer or enterprise?
2. Is the category already playful or very serious?
3. Does the product need warmth, trust, urgency, or craft?
4. Can illustration be maintained across future pages?
5. Is a custom typeface available?
6. Is there a product UI proof asset?
7. What one color can own action?

Then choose:

- illustration if category needs emotional reframing
- typography if brand needs authority
- color if product must stay simple
- mode transition if hero needs drama and body needs clarity
- material if audience is technical/premium

```tsx
    </section>
  );
}
```

Tailwind guardrails:

- Use tokens for brand colors and radii.
- Do not create arbitrary one-off colors in every component.
- Keep display type out of dense UI.
- Put illustration colors in separate semantic names.
- Use `shadow-brand-ring` instead of generic shadows for flat card systems.

## 19. QUALITY CHECKLIST

### 19.1 Identity

- [ ] The main expressive lever is clear.
- [ ] The support lever is clear.
- [ ] The brand appears beyond the hero.
- [ ] Repetition rules are defined.
- [ ] The system fits the product category.

### 19.2 Color

- [ ] Primary accent has a strict role.
- [ ] Forbidden accent roles are known.
- [ ] Semantic colors are separate.
- [ ] Illustration colors do not confuse UI colors.
- [ ] Palette works in dark/light contexts.

### 19.3 Typography

- [ ] Display font has a defined role.
- [ ] UI font remains readable.
- [ ] Tracking is intentional.
- [ ] Mobile wrapping is controlled.
- [ ] Decorative type is not used for dense UI.

### 19.4 Illustration / Graphics

- [ ] Illustration vocabulary is defined.
- [ ] Characters/props have consistent anatomy.
- [ ] Color count is controlled.
- [ ] Graphics do not replace product proof.
- [ ] Placement rules are repeatable.

### 19.5 Product Proof

- [ ] Hero has or leads to a real product object.
- [ ] Screenshots/mockups use brand frames.
- [ ] Product UI is readable.
- [ ] Feature cards communicate actual value.
- [ ] Brand style does not obscure workflow.

### 19.6 Components

- [ ] Buttons follow radius/action rules.
- [ ] Cards follow the chosen elevation method.
- [ ] Nav includes micro-brand details if appropriate.
- [ ] Labels and badges use consistent type.
- [ ] CTA is obvious.

### 19.7 Motion

- [ ] Motion personality is chosen.
- [ ] Timing scale is consistent.
- [ ] Reduced motion is respected.
- [ ] Decorative motion does not distract.
- [ ] Shimmer/character movement is rare.

### 19.8 Anti-Slop

- [ ] No random mascots.
- [ ] No extra saturated accent colors.
- [ ] No generic SaaS bento with decorative skin.
- [ ] No hero-only identity.
- [ ] No product proof from another visual system.

<a id="prompting"></a>
## 20. PROMPTING GUIDE

### 20.1 General Expressive Brand Prompt

Create an expressive brand design system for a modern product website. Start
with one primary brand lever and one supporting lever. Make the identity
repeatable across hero, navigation, cards, product frames, buttons, labels, and
motion. Avoid random decorative elements, too many accent colors, generic SaaS
gradients, and expressive visuals that do not support product proof.

### 20.2 Storybook Fintech Prompt

Design a warm cream fintech website with a disciplined illustrated character
system. Use a #fbfaf9 canvas, graphite text, flat cream cards with inset warm
stone rings, a near-black pill CTA, and expressive flat characters with blob
bodies, stick limbs, dot eyes, and a controlled primary-color palette. Keep the
product UI serious and contained in rounded mockups.

### 20.3 Private Editorial Prompt

Design a private personal-knowledge product with editorial serif headlines,
orange uppercase eyebrow labels, orange nav dots, warm orange gradient
atmosphere, pastel feature cards, and clean product screenshot frames. Use
orange as outline/label/atmosphere, not filled CTA. Keep the tone calm,
reflective, and personal.

### 20.4 Electric Infrastructure Prompt

Design a technical infrastructure landing page with a dramatic deep navy to
electric-blue hero, a dot-matrix network/globe visual, one chartreuse primary
CTA, pill buttons, tight custom sans UI, high-craft display headings, and an
immediate transition into a light #f8f9fc dashboard proof section with
blue-tinted shadows.

### 20.5 Bold Finance Prompt

Design a bold financial utility website with high-contrast Night Sky dark
sections, huge condensed/expanded typography, one vivid yellow primary action
color, rounded 24px cards, and clean white content sections. The brand should
feel direct, confident, useful, and trustworthy without extra decorative
colors.

### 20.6 Warm Workshop Prompt

Design a developer platform website with a warm obsidian #151515 canvas,
parchment #e9ebdf text, huge light compressed sans headlines, dark slab
surfaces instead of shadows, muted teal washes, a warm amber-rust hero
atmosphere, and binary shape grammar: pill CTAs or square buttons, no medium
rounded compromise.

### 20.7 Negative Prompt

Do not use random mascot stickers, stock illustrations, generic purple/blue
SaaS gradients, multiple saturated CTA colors, decorative type in form labels,
low-contrast playful palettes, product screenshots that do not match the brand
system, or personality that appears only in the hero.

<a id="quick-reference"></a>
## 21. QUICK REFERENCE

### Pick The Archetype

| Need | Archetype |
| --- | --- |
| approachable fintech/crypto | Illustrated Fintech Storybook |
| private memory/knowledge | Private Memory Editorial |
| infrastructure/devops | Electric Infrastructure Signal |
| finance utility | Bold Finance Utility |
| developer platform | Warm Obsidian Workshop |

### Pick The Main Lever

| Lever | Use when |
| --- | --- |
| Illustration | category needs warmth or emotional reframing |
| Typography | brand needs authority and distinction |
| Single accent | product must stay simple and usable |
| Mode transition | hero needs drama, body needs clarity |
| Material | audience is technical or premium |
| Motion | brand personality should feel alive |

### Do

- define the expressive system
- repeat brand details at multiple scales
- protect product readability
- use one primary accent role
- make component grammar strict
- build real product proof
- separate brand colors from semantic states

### Don't

- add random colors
- use mascots as stickers
- overuse decorative fonts
- hide product behind personality
- mix radius systems randomly
- copy a hero style without the supporting component rules
- let the brand disappear after the first viewport

### Final Test

Remove the logo. If the page still feels identifiable, the expressive brand is
working. Remove the hero. If the cards, buttons, labels, typography, and
product frames still feel like the same brand, the system is real.
