# Legend - Editorial Minimal Source Notes

## Core read

Legend is the architectural blueprint variant. It is nearly achromatic, precise,
and slightly technical. The system uses a warm gray canvas, restrained black
type, lightweight components, and a vivid violet accent that behaves like a
switched-on data point rather than a marketing color.

This reference is useful because it shows how editorial minimalism can become
more architectural and product-like without turning into generic SaaS. The page
has measurable geometry: max width, element gap, mono coordinates, and specific
radius roles.

## Typography

- Primary: knapp or Inter fallback.
- Weights: 400, 500.
- Sizes: 12, 16, 20, 22, 44, 56.
- Line heights: 1.00 to 1.50.
- Tracking: about -0.01em.
- Mono: diatypeMono or Space Mono.
- Mono sizes: 11 and 13.
- Mono tracking: around 0.06em.
- Mono role: coordinates, technical details, subtle nav.

## Structure

- Max width: around 1416px.
- Density: comfortable.
- Base unit: 4px.
- Section gap: 68px.
- Card padding: 20px.
- Element gap: 20px.
- Buttons/general radius: 4px.
- Nav items: 12px.
- Cards: 32px.

## Component lessons

1. Violet edge accent
   - Use for outline, focus, tags, and data.
   - Do not promote it to every CTA.

2. Architectural card
   - Large 32px radius but minimal fill.
   - Feels like a slab, not a soft SaaS card.

3. Mono coordinate
   - Small, letter-spaced, structural.
   - Adds blueprint feeling without visual clutter.

4. Muted gradient section
   - Desaturated, naturalistic, and rare.
   - Used as tonal depth, not colorful hero decoration.

5. Dark contrast block
   - Deep Gray card or panel.
   - Creates editorial rhythm against warm gray canvas.

## Design lessons

- Minimalism can feel architectural when measurements are visible.
- A single violet accent can read as technical if used on edges and tags.
- Large card radius can work when everything else remains stark.
- Mono details can imply coordinates, systems, and precision.
- Warm gray canvas is more distinctive than sterile white.

## Anti-patterns

- Do not introduce new saturated colors.
- Do not use heavy drop shadows.
- Do not use generic type scale values.
- Do not use the violet as a broad fill everywhere.
- Do not make the gradient colorful or soft-SaaS.
- Do not mix many radii beyond the defined roles.
