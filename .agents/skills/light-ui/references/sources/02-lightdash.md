# Lightdash — Light UI Source Notes

Source: https://styles.refero.design/style/d0f65d12-a8e6-4631-99f7-bb7cdcd5b6c5
Original URL: https://www.lightdash.com
Category scan: Light UI
Theme: light
North star: Codebase blueprint on frosted glass.

## Essence

Lightdash is a precise technical light UI. It combines white and soft gray
surfaces with deep neutral text, an electric violet accent, robust cards, and
specialized typography. It feels like a codebase blueprint translated into a
friendly analytics product. Depth is present, but light and controlled.

## Typography

- Britti Sans Trial Semibold: display and major headings.
- Britti Sans Medium: subheadings and prominent labels.
- Inter: body, navigation, fields, UI.
- IBM Plex Mono: code and technical readouts.
- Micro 5: rare decorative digital display.
- Display scale reaches `76px`, line-height `0.9`.

## Layout

- Contained centered layout.
- Sticky top bar.
- Alternating white and lava-cloud section surfaces.
- Two-column product/explanation sections.
- Spacing uses `40-64px` section gaps.

## Components

- Primary button: Electric Violet fill, white text, `8px` radius.
- Secondary ghost button: transparent, midnight border, `8px` radius.
- Interactive chip: light fill, `26px` radius.
- Elevated feature card: white, `20px` radius, layered subtle shadow.
- Simple info card: white, `12px` radius, 1px shadow/border.
- Dark input: deep indigo background, white text, sharp or minimal radius.

## Reusable Lessons

- Light SaaS can feel technical through type and subtle blueprint surfaces.
- Electric Violet should mark interaction only.
- Use mono type for code/data, not decoration.
- Cards can have depth if shadows are extremely soft and systematic.

## Anti-Patterns

- Do not add new saturated hues.
- Do not use violet for body text or large blocks.
- Do not replace Britti/Inter hierarchy with generic defaults.
- Do not overuse dark sections in a light-mode system.
