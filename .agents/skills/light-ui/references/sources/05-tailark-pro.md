# Tailark Pro — Light UI Source Notes

Source: https://styles.refero.design/style/ee8698fd-bb1e-4813-9571-5db39e508542
Original URL: https://pro.tailark.com/illustrations
Category scan: Light UI
Theme: light
North star: Gallery of Digital Artifacts.

## Essence

Tailark Pro is a light UI gallery system: near-white walls, subtle shadows,
neutral sans typography, mono precision labels, varied corner radii, and small
vivid accents used inside components. It feels like an exhibition of digital
artifacts rather than a flat documentation page. The design is airy but still
component-rich.

## Typography

- `ui-sans-serif`: primary body, nav, buttons, headings.
- Geist Mono: code snippets, command lines, technical labels.
- Mulish: softer descriptive body copy in limited places.
- Display size around `30px`, body `14px`, caption `10px`.

## Layout

- Centered contained layout around 1200px.
- Sticky compact header.
- Modular gallery grids.
- Uses 2-column and 3-column card arrangements.
- Alternates white and silver-gray/tinted card backgrounds.
- Element gap: `8px`; section gap: `48px`.

## Components

- Default button: transparent, subtle gray border, `6px` radius.
- Illustration card: pale translucent surface, sometimes `0px` radius.
- Elevated card: white, `16px` radius, subtle layered shadow.
- Top-rounded cards: `32-40px` top radius and square bottom.
- Input: translucent fill, `4px` radius.
- File type badges: mono labels with small colored states.

## Reusable Lessons

- Light UI can be rich if surface depth is subtle and systematic.
- Varying radius can create rhythm when carefully controlled.
- Mono labels make component galleries feel engineered.
- Accents should live in artifacts and active states, not everywhere.

## Anti-Patterns

- Do not use heavy shadows.
- Do not add opaque borders everywhere.
- Do not use many vibrant colors at once outside illustrations.
- Do not default to one radius for every component.
