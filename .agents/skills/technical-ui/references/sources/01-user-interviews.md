# User Interviews — Technical UI Source Notes

Source: https://styles.refero.design/style/376baf20-9ace-405d-bf4a-086016f2b1e3
Original URL: https://www.userinterviews.com
Category scan: Technical UI
Theme: light
North star: Teal-accented architectural blueprint on pristine parchment.

## Essence

User Interviews is a friendly technical UI. It uses a light architectural canvas,
serif/sans hierarchy, teal action color, large rounded components, and generous
section spacing. The system feels organized and operational without becoming
cold. It is especially useful for research tools, workflow products, recruiting
platforms, and user-facing technical SaaS that needs trust and approachability.

## Typography

- `sofia-pro`: UI, body, navigation, buttons, labels.
- `p22-mackinac-pro`: headings and major statements.
- `P 22 Mackinac italic`: emphatic callouts.
- `IBM Plex Mono`: data, code, structured readouts.
- Display: `64px`, line-height `1.16`.
- Body: `14px`, line-height `1.57`.

## Layout

- Contained centered page, roughly 1200px.
- Full-width hero with centered headline.
- Alternating Canvas Ice and Cloud Frost sections.
- 2-column text + image/product layouts.
- Multi-column card grids.
- Section gap around `88px`.
- Sticky nav with clear sign-up/sign-in actions.

## Components

- Primary filled button: Oceanic Teal, white text, `48px` radius.
- Secondary filled button: Emerald Green, white text, `48px` radius.
- Ghost button: transparent, black outline, `88px` radius.
- Outlined tags: thin colored borders, `100px` radius.
- Nav links: black text with Deep Teal underline/border active state.
- Info banner: Cloud Frost background, black text.

## Reusable Lessons

- Technical UI can be rounded and warm without losing clarity.
- Serif headings can humanize a workflow platform.
- Teal works well as a trust-oriented action color.
- Mono should be reserved for structured data, not body copy.

## Anti-Patterns

- Do not use sharp corners.
- Do not add random saturated colors.
- Do not use mono for general text.
- Do not over-border every interactive element.
