# Archive migration SPA vers Next

Ces fichiers sont conserves comme historique de decision et preuve de migration.

Ils ne sont plus des consignes operationnelles actives. L'etat courant du projet est documente dans:

- `NEXT_NATIVE_ARCHITECTURE_BASELINE.md`
- `AGENTS.md`
- `context.md`
- `mapV2.md`

Ne pas utiliser ces rapports pour reintroduire `ClientApp`, `src/app.jsx`, `src/Router.jsx`, un routing `setView`, ou un fallback SPA public.

Les rapports archives peuvent rester utiles pour comprendre:

- pourquoi la migration Next App Router a ete lancee;
- quels gates ont prouve la sortie du shell SPA;
- quelles decisions SEO/SSR/SSG/ISR ont ete prises;
- quels ecarts historiques ont deja ete traites.
